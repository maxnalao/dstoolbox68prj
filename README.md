# Software Development Methodologies

ปัจจุบันมีวิธีการพัฒนา Software ที่นิยมหลัก ๆ ดังนี้

## 1. Waterfall Model
- ทำงานเป็นขั้นตอนเรียงลำดับ
- เหมาะกับโปรเจกต์ที่ Requirement ชัดเจน
- เข้าใจง่าย แต่ยืดหยุ่นต่ำ

## 2. V-Model
- คล้าย Waterfall แต่เพิ่มการทดสอบในทุกขั้น
- ช่วยควบคุมคุณภาพดี เหมาะกับงานสำคัญ เช่น ระบบแพทย์

## 3. Iterative Model
- พัฒนาเป็นรอบ ๆ ทยอยเพิ่มฟีเจอร์
- ลดความเสี่ยงและแก้ไขได้ระหว่างทาง

## 4. Spiral Model
- เน้นประเมินความเสี่ยงทุกรอบของการพัฒนา
- ใช้กับโปรเจกต์ใหญ่และซับซ้อน

## 5. Agile
- ยืดหยุ่นสูง ทำงานเร็ว ปรับ Requirement ได้ตลอด
- เหมาะกับ Startup และโปรเจกต์ที่เปลี่ยนบ่อย

## 6. Scrum (Agile Framework)
- ใช้รอบการทำงานแบบ Sprint (1–4 สัปดาห์)
- มีบทบาท PO, SM, Dev Team
- จัดการงานเป็นระบบและส่งงานได้ต่อเนื่อง


## ⭐ วิธีการพัฒนา Software ที่นิยมที่สุดปัจจุบัน

### 1. Agile (โดยเฉพาะ Scrum) – นิยมที่สุด
Agile เป็นแนวทางการพัฒนาซอฟต์แวร์ที่ยืดหยุ่นและตอบสนองต่อการเปลี่ยนแปลงได้ดี  
โดยใช้รอบการทำงานสั้น ๆ (Sprint) ทำให้สามารถส่งงานได้เร็วและรับ Feedback ได้ทันที

**เหตุผลที่ Agile / Scrum นิยมมากที่สุด**
- ปรับเปลี่ยน Requirement ได้ง่าย
- ส่งงานเป็นรอบสั้น ทำให้ลดความเสี่ยง
- ทีมทำงานร่วมกันง่ายและเร็วขึ้น
- ลูกค้าเห็นความคืบหน้าอย่างต่อเนื่อง
- ใช้ในบริษัทใหญ่ทั่วโลก เช่น Google, Microsoft, Meta, Amazon
